# Skill: Дайджест новостей с Хабра

## ВАЖНО: разрешённые инструменты

**Используй ТОЛЬКО эти инструменты:**
- `search_habr` — поиск статей на Хабре
- `fetch_article` — загрузка текста статьи
- `memory` — чтение и запись памяти для дедупликации

**НЕ используй:** browser, execute_code, terminal, web_search и любые другие инструменты.
Если `search_habr` не вернул результатов — сообщи об этом пользователю, не пробуй другие инструменты.

---

## Когда применять

Применяй этот навык, когда пользователь просит собрать дайджест, обзор или сводку новостей по теме за какой-либо период.

Примеры запросов:
- «собери дайджест по теме AI за последнюю неделю»
- «что нового по Rust за месяц?»
- «сделай обзор новостей про Docker за 3 дня»

---

## Шаг 1. Извлеки тему и период из запроса

Разбери запрос пользователя на два параметра:

**Тема (`query`)** — ключевые слова для поиска. Примеры:
- «AI / ML» → `"искусственный интеллект OR machine learning OR LLM"`
- «Rust» → `"Rust"`
- «безопасность» → `"информационная безопасность"`

**Период (`period`)** — преобразуй в формат MCP:
- «за сутки», «за день», «за 24 часа» → `"24h"`
- «за неделю», «за последнюю неделю», «за 7 дней» → `"7d"`
- «за две недели» → `"14d"`
- «за месяц» → `"1m"`
- Если период не указан — используй `"7d"` по умолчанию

---

## Шаг 2. Найди статьи (search_habr)

Вызови `search_habr` с параметрами из шага 1, `limit: 15`.

Если результатов мало (< 3) — сделай максимум **один дополнительный** вызов `search_habr` с переформулированным запросом (например, замени русское слово на английское). Больше двух вызовов `search_habr` делать нельзя.

---

## Шаг 3. Дедупликация через память Hermes

В начале каждой сессии в твой системный промпт уже вшита секция MEMORY.md с тем, что ты сохранял раньше. Проверь её на наличие записей с тегом `digest-seen:`.

- Если URL статьи уже есть в `digest-seen:` — **исключи статью из рассмотрения**.
- Статьи, которых нет в памяти — оставь.

Это позволяет не повторять материалы из прошлых дайджестов.

---

## Шаг 4. Загрузи и аннотируй каждую статью (fetch_article)

Из найденных (и не исключённых дедупликацией) статей отбери **5–8 наиболее релевантных** по заголовку и snippet.

Для каждой отобранной статьи:
1. Вызови `fetch_article(url)`
2. Прочитай текст
3. Напиши **аннотацию TL;DR на 2–4 предложения**: что нового, зачем это важно, в чём суть

Критерии отбора:
- Соответствие теме запроса
- Свежесть (предпочитай более новые)
- Уникальность (не дублируй похожие материалы)
- Содержательность (пропускай пустые анонсы без деталей)

---

## Шаг 5. Напиши итоговую статью-дайджест

Собери все аннотации в **цельную обзорную статью** со следующей структурой:

```
**Дайджест: [Тема] — [Период]**

[Вводка — 2–3 предложения: о чём этот дайджест, сколько материалов, общий тренд]

---

**[Заголовок статьи 1]**
[Аннотация 2–4 предложения]
[Ссылка](url)

**[Заголовок статьи 2]**
[Аннотация 2–4 предложения]
[Ссылка](url)

... (все отобранные статьи)

---

**Вывод**
[2–4 предложения: общий вывод, ключевые тренды, что стоит запомнить]
```

---

## Шаг 6. Сохрани URL статей в память (дедупликация)

После отправки дайджеста **молча** (без каких-либо слов пользователю) вызови инструмент `memory`:

- `action` = `"add"`
- `target` = `"memory"`
- `content` = `"digest-seen: <url1>, <url2> — тема: <тема>, дата: <YYYY-MM-DD>"`

Не выводи JSON, не упоминай этот шаг в тексте ответа, не добавляй секцию «Сохранение» — просто вызови инструмент и всё.

---

## Правила тона и языка

- Язык: **русский**, нейтрально-технический
- Не пиши «Я нашёл» / «Я собрал» — пиши напрямую о содержании
- Не копируй целые абзацы из статей — **перефразируй**
- Факты проверяй по тексту статьи, не додумывай
- Избегай оценочных суждений («отличная статья», «плохое решение»)
- Если статья на иностранном языке — аннотируй на русском

---

## Формат ответа в Telegram

Telegram поддерживает Markdown. Используй:
- `**жирный**` для заголовков статей
- `[текст](url)` для ссылок
- `---` как разделитель
- Не используй HTML-теги

Если итоговый текст длиннее 4000 символов — раздели на 2–3 сообщения, каждое с логическим завершением.

---

## Обработка ошибок

- Если `search_habr` вернул менее 3 релевантных статей — сообщи пользователю и предложи расширить период или уточнить запрос.
- Если все найденные статьи оказались в памяти (дедупликация) — сообщи: «Все найденные статьи по теме "[тема]" за [период] уже были в предыдущих дайджестах. Попробуй запросить более длинный период.»
- Если `search_habr` вернул пустой массив — сообщи: «По теме "[тема]" за [период] на Хабре ничего не найдено.»
- Если `fetch_article` вернул ошибку — пропусти статью, продолжи с остальными.
- Если успешно загружено менее 3 статей — предупреди пользователя.
